PiHPe

Marcel Varkevisser | LinkedIn | GitHub

What

A bit of PHP to help me create simple websites. It allows me to:

  • isolate content in separate files A
  • have relative paths and links sorted B
  • publish static versions (i.e. GitHub), too C
  • use HTML/CSS/JS, with a bit of PHP
  • have proper syntax highlighting

How

message.php A

Carefully spreading content over different files, makes it much nicer to work on... 

<? ob_start(); ?> 
   <p>A bit of PHP to help me create...</p>
<? $body = ob_get_clean(); ?>
Each file contains all HTML, CSS, and JavaScript for one piece of content. External stylesheets and -scripts are also declared here. And when content is combined on a page, those externals are loaded only once by PiHPe.

header.php

...and easy to reuse content on multiple pages: 

<? ob_start(); ?> 
   <style>
      h1 { line-height: 100vh; }
   </style>
<? $style = ob_get_clean(); ?>
<? ob_start(); ?> 
   <h1>PiHPe</h1>
<? $body = ob_get_clean(); ?>

/_pihpe/paths.php B

I give everything a unique name: 

$paths = [
   'message' => '/message.php',
   'header'  => '/header.php',
   'layout'  => '/_php/layout.php',
   'favicon' => '/favicon.png'
];

Remember DRY, and simply include URL's with a but of PHP: 
<link rel="icon" href="<?= url('favicon'); ?>">

index.php

Decide what to add to a page, and which layout to use: 

parse([
   'header',
   'message'
]);

include($root . url('layout'));

layout.php

<main>
   <? insert($body); ?>
</main>

/_pihpe/build.php C

Define what to include in a build. Browse to /[pihpe]/build to create a static version: 

build([
   '/index.php'
]);
